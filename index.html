Git Essencial

O essencial sobre Git

Todos os comandos básicos que você precisa para realizar a maioria das atividades que eventualmente você fará no Git. Ao final desse capítulo você deverá ser capaz de configurar e inicializar um repositório, começar e parar o monitoramento de arquivos, além de selecionar e consolidar (fazer commit) alterações. Também vamos mostrar a você como configurar o Git para ignorar certos tipos de arquivos e padrões de arquivos, como desfazer enganos de forma rápida e fácil, como pesquisar o histórico do seu projeto e visualizar alterações entre commits e como enviar e obter arquivos a partir de repositórios remotos.

Comandos

git init: Initializa repositório vazio em um diretório existente. Ex: /Users/wallacesales/Sites/git_essencial/.git/

git status: Verifica o estado dos seus arquivos.

touch <nome do arquivo>: Cria um arquivo no repositório.

Add

git add <nome do arquivo>: Diz explicitamente ao Git para monitorar um arquivo. Retira o arquivo do working directory (diretório de trabalho) e manda para staging area (área intermediária). Um commit só pode ser feito caso o arquivos criado/alterado esteja adicionado na staging area.

git add -A: -A faz o git adicionar todos os arquivos.

git add .: Faz o git adicionar todos os arquivos.

git add *.txt: *.txt faz o git adicionar todos os arquivos de texto com extensão .txt. Pode ser usada qualquer extensão.

git rm <nome do arquivo>: Para remover um arquivo do Git, é preciso removê-lo da staging area e, depois, fazer o commit. O comando git rm também remove o arquivo do working directory.

git mv <nome do arquivo>: Altera o nome de um arquivo do repositório.

git checkout <nome do arquivo>: Desfaz as modificações no arquivo que ainda não está em ambiente de stage.

git reset HEAD <nome do arquivo>: Remove o arquivo alterado do espaço de stage para permitir o uso do comando git checkout para desfazer as modificações feitas no arquivo.

Commit

git commit -m <nome do arquivo>: Envia diretório(s) e/ou arquivo(s) ao repositório. vale lembrar que os commits são todos locais.

git commit -a -m: Envia arquivo(s) ao repositório sem precisar do comando git add para adicionar à stage area, pois o -a já faz esse papel.

git commit --amend -m: Edita um commit já feito. Não cria um novo commit, mas adiciona as alterações. Contudo, o hash (chave) muda.

git revert <nome do arquivo>: Remove um commit específico da lista de commits.

Reset: existem três formas de fazer o reset: soft, mixed e hard.

git reset --soft <commit>: Basicamente só altera a referência de master (move apenas o ponteiro HEAD para algum outro commit), sem alterar a área de stage ou o diretório de working. Bom para fazer mudanças num commit existente.

git reset --mixed <commit>: É o tipo default. Essa opção, além de mover o ponteiro HEAD, faz com que a área de stage contenha o mesmo snapshot do commit para o qual o ponteiro HEAD foi movido, porém não afeta o diretório de working.

git reset --hard <commit>: É o mais perigoso tipo de reset. Ele não apenas descarta as alterações na área de stage como também reverte todas as alterações no diretório de working para o estado do commit que foi especificado no comando.

Log

git log: Serve para analisar todos os commits realizados no repositório. Para sair basta pressionar a tecla q. Exibe: mensagem, hash do commit, autor e data.

git log -p: Exibe mais detalhes dos commits realizados no repositório. Para sair basta pressionar a tecla q. Exibe: informações do diff (o que foi feito em cada arquivo que foi alterado), mensagem, hash do commit, autor e data.

git log -p -n: -n é o número de commits desejável. Exemplo: git log -p -1.

git log -pretty=oneline <nome do arquivo>: Mostra apenas o hash do commit e a mensagem.

git log --online --decorate: Condensa cada commit em uma única linha. O sinalizador --decorate faz o git log exibir todas as referências (por exemplo, branches (ramificações), tags, etc.) que apontam para cada commit.

git log --online --decorate --all --graph: Condensa cada commit e exibe em forma de gráfico.

Diff

git diff: Exibe as diferenças num determinado arquivo que está no working directory.

git diff --staged: Exibe as diferenças num determinado arquivo que está na staging area.

Tag

git tag: O Git tem a capacidade de marcar pontos específicos na história como sendo importantes. Normalmente, as pessoas usam essa funcionalidade para marcar pontos de lançamento. Exemplo: git tag -a v1.0 -m "Versão 1.0.".

git tag -a <tag> -m: Cria a tag no momento atual do repositório. Exemplo: git tag -a v1.0 -m "Versão 1.0.".

git tag -a <tag + hash do commit> -m: Cria a tag num momento anterior do repositório. Exemplo: git tag -a v0.0 2b1e9f027d29882acd2b9407eb99a97003c13aef -m "Versão 0.0.".

git show <tag>: Exibe os detalhes da tag. Exemplo: git show v1.0.

git checkout <tag>: Troca o repositório (versão do sistema/projeto) para o status em que a tag foi criada. Exemplo: git checkout v0.0.

git tag -d <tag>: Exclui a tag. Exemplo: git tag -d v0.0.

Branch, ou ramificaçnao.

git branch <branch>: Cria um branch (ramificação). O uso de tags para controlar versão e/ou criar ambientes diferentes de desenvolvimento de um projeto pode trazer transtornos. O melhor é fazer o uso de branches (ramificações), pois permite trabalhar com varias ramificações diferentes sem que uma interfira na outra.

git checkout <branch>: Faz a transição dos arquivos do branch master para o novo branch criado.

git checkout -b <branch>: Uma forma simplificada para criar o branch e fazer a transição dos arquivos do branch master para ele.

git branch -d <branch>: Deleta o branch.

Stash é uma área, onde é possível armazenar código, sem a necessidade de realizar um commit,permitindo mudar de brach ou realizar outras operações de forma organizada.

git stash save "<flag>": Envia o arquivo para o stash retirando o da staging area. Após o comando, o git exibe mensagem como: stash@{0}: On feature4: h.txt.

git stash apply <stash>: Devolve o arquivo para a staging area, mas ainda continua no stash. Ex: git stash apply stash@{0}.

git stash drop <stash>: Apaga o arquivo do stash. Ex: git stash drop stash@{0}.

git stash pop: Devolve o arquivo para a staging area e apaga o arquivo do stash. Ex: git stash pop.

git stash branch <branch>: Cria um novo branch a partir de um stash.

Push, ou empurrar, é a ação de atualizar uma referência remota a partir de uma referência local, enviando os objetos necessários para satisfazer as referências atualizadas.

git remote add <nome>: Adiciona o repositório remoto. Ex: git remote add origin https://github.com/rodrigobranas/gitpresentation

git remote: Exibe o repositório remoto configurado.

git remote -v: Exibe o repositório remoto configurado com o endereço.

git push: Envia os commits ao repositório remoto.

git push origin master: Envia os commits ao repositório remoto, origin, a partir do master.

Pull, ou puxar, é a ação de atualizar uma referência local a partir de uma referência remote, trazendo os objetos necessários para satisfazer as referências atualizadas. O cuidado que se deve ter é porque o git pull faz merge automaticamente.

git pull: Faz a operação inversa do push.

git pull origin master: Recebe os commits ao repositório remoto, origin.

git fetch: Faz o pull pela metade. Ele só traz as atualizações e aplica. Não faz merge.

git diff master origin/master: Exibe as diferenças entre master e origin/master.

git merge origin/master: Faz a mesclagem dos arquivos com o origin. Fazer apenas se o diff não apresentar nenhum conflito.

git clone <endereço de origem> <destino>: Clona um repositório remoto.
